# 야간 자율 개발 로그 (2026-06-09 밤 ~ 06-10 새벽)

## 한 줄 요약
1688 상품의 **상세페이지 이미지를 수집하고, 이미지 속 중국어를 한국어로 자동 번역**해
실제 한국 상세페이지처럼 만드는 파이프라인을 **로컬·무료(번역만 Haiku 소액)** 로 구축·검증했습니다.

---

## 만든 것 (전부 검증 완료)

### 이미지 번역 엔진
- **`scripts/translate_image.py`** — 이미지 속 중국어→한국어 번역 렌더링
  - 스택: **RapidOCR**(검출, torch 불필요) → **Haiku**(자연 번역) → **cv2.inpaint**(원문 제거) → **Pillow+맑은고딕**(한글 렌더)
  - 종횡비로 세로글자 감지, 채도 높은 단색 뱃지만 색 채움·나머지는 inpaint 후 흰글자, 글자색 매칭, 자동 줄맞춤
  - 단일: `python scripts/translate_image.py <in> <out> --cache _out/cache.json`
  - 배치: `python scripts/translate_image.py <폴더in> <폴더out> --batch --cache _out/cache.json` (OCR 모델 1회 로드)

### 상세 이미지 스크래핑
- **`scripts/scrape-detail.mjs <offerId>`** — 상세페이지를 끝까지 스크롤해 lazy 이미지(data-src 포함) 수집. 상품당 5~25장

### 종단 파이프라인
- **`scripts/process-product-images.mjs <offerId>`** — 스크래핑→배치번역→Storage 업로드→DB 갱신
  - Storage: `product-images/<id>/detail/NN.jpg`
  - DB: `products.images = [대표이미지, ...번역된 상세컷]`
- **`scripts/process-batch.mjs [N]`** — 아직 상세컷 없는 상품 N개를 한 번에

### 이미지 재호스팅 (앞서 완료)
- **`scripts/rehost-images.mjs`** — 원본 CDN → Supabase Storage 이전, DB URL 교체

## 번역 품질 (실제 결과 예시)
| 원문(중국어) | 번역(한국어) |
|---|---|
| 纯欲睡裙 | 청순섹시 슬립 드레스 |
| 内置杯垫免穿义胸 | 패드 내장, 노브라 가능 |
| 聚拢美胸 | 가슴 모아주는 |
| 浅蓝/粉色/卡其 … | 연파랑/핑크/카키 … (색상표 7/7) |
| 欢走光韩口点 (OCR 깨짐) | 비침 방지 스타일링 (의미 복원) |

- 프롬프트에 패션 용어 가이드 + OCR 오인식 보정 지시를 넣어 자연스러움↑
- 색상표 이미지: 7개 색상 전부 단일 줄 한국어로 깔끔 교체

## 한계 (남은 개선점)
1. **장식체/저대비 글자 OCR 오독** 가끔 (번역단에서 일부 보정되나 완벽치 않음)
2. 콜아웃이 빽빽한 이미지에서 한글이 약간 겹칠 수 있음

## 사용법 (다음에 바로)
```bash
# 한 상품 종단 처리
node scripts/process-product-images.mjs <offerId>

# 여러 상품 일괄
node scripts/process-batch.mjs 10

# 결과 DB 검증
node scripts/verify-collection.mjs --since 600
```
